Assumptions:

Identity is mediated via ETH‑compatible signing on a permissioned/L2 ledger.

Core threats
T1 – Malicious page/script triggers stealth ETH signing
Goal: obtain signatures that can be replayed to impersonate an augmented user or bind their DID without consent.

1. Feature vector design (per time step)
Each time step ttt corresponds to a small window, e.g., 5–10 seconds or “per signing decision.” For each ttt, the browser/agent derives a feature vector xt∈Rdx_t \in \mathbb{R}^dxt∈Rd with the following fields:
sign_count_window
Meaning: Number of signing attempts in the last window (5–10s).
Range: 0–20 (clip at 20).
Normalization: v=min⁡(c,20)/20v = \min(c, 20) / 20v=min(c,20)/20.

Meaning: Count of distinct domains requesting signatures in the window.
Range: 0–10 (clip at 10).
Normalization: v=min⁡(d,10)/10v = \min(d, 10) / 10v=min(d,10)/10.
domain_trust_score
Meaning: Score of current domain based on safelist/blocklist (e.g., 1.0 = whitelisted gov/ALN, 0.5 = neutral, 0.0 = unknown/suspicious).
Range: 0.0–1.0 (already normalized).
user_interaction_ratio
Meaning: Fraction of sign requests preceded by explicit user actions (click/keypress/biometric) within last X seconds.
Range: 0.0–1.0.

Meaning: Time (seconds) since last real user gesture.

2. AEFD (Autoencoder Feature Denoising) configuration
Goal: compress noisy behavioral vectors into a smaller latent representation and smooth out outliers before sequence modeling.
Input dimension: 12.
Encoder: 12 → 8 → 4 units (ReLU activations).
Latent vector: 4‑dimensional ztz_tzt.
Decoder: 4 → 8 → 12 units (ReLU + linear output).
Training objective: Minimize reconstruction loss L=∥xt−x^t∥22L = \|x_t - \hat{x}_t\|_2^2L=∥xt−x^t∥22 on known “clean” behavior logs.
Denoising operation at runtime:
Feed normalized xtx_txt to encoder to get ztz_tzt.
Optionally reconstruct x^t\hat{x}_tx^t; if reconstruction error > threshold, mark the step as anomalous even before TANN.
Pass only ztz_tzt (4‑D) forward to the transformer.
This reduces dimensionality and noise, and ensures the sequence model works over compact vectors.
3. Sequence configuration for TANN
Use the latent sequence {zt−L+1,…,zt}\{z_{t-L+1}, \dots, z_t\}{zt−L+1,…,zt} as input, where:
Window length L: 32 steps (typical) or 64 for more context.
Example: if each step is 10 seconds, 32 steps cover ~5 minutes, enough to capture bursts and gradual drifts.
Input shape:
Sequence length: 32.
Feature dimension: 4.
So tensor: [batch_size, 32, 4].
Transformer architecture (lightweight)
Embedding:
Project 4‑D input to 16‑D model space: dense layer 4 → 16.
Add positional encoding (sin/cos) for positions 0…31.
Encoder blocks:
2 encoder layers.
Each with:
Multi‑Head Self‑Attention (MHA) with 2–4 heads (e.g., 4 heads, head dim=4).
Feedforward network: 16 → 32 → 16 (ReLU).
LayerNorm + residuals.
Output:
Pool across time (e.g., mean pooling) → 16‑D vector.
Final classifier: 16 → 1 (sigmoid), output p_anomaly ∈ [0,1].
Decision rule
If p_anomaly ≥ θ, block signing and raise alert.
Threshold θ can be tuned; a typical starting point is 0.9 on validation data, or adjust based on target false positive rate.
To reproduce:
Collect benign sequences of (x_t) on test machines.
Train AE on those; freeze encoder for runtime.
Generate z_t and sequences (32 steps) → train transformer classifier on labeled benign vs simulated attack sequences (scripted mass signing, unknown domains, low user_interaction_ratio, etc.).
4. Integration with ETH signing flow
At each signing decision:
Update features based on recent behavior → compute x_t.
Denoise/compress:
z_t = Encoder(x_t).
Append z_t to rolling buffer of last L steps; if buffer < L, pad with zeros or initial baseline.
Run TANN on sequence → p_anomaly.
If p_anomaly < θ and policy checks (domain safelist, consent prompt) pass:
Proceed to challenge/response flow (Augmented‑ID signing).
Else:
Block signing, log event, optionally require explicit re‑auth or show a security prompt.
This keeps the guard fast enough for real‑time use at the browser/edge while enforcing strong behavior‑based anomaly detection.

4. TS guard logic (sequence + policy)

// guard.ts
import { FeatureVector, TrimSecConfig, RiskResult, SignContext } from "./index";
import { buildFeatureVector } from "./features";
import { encodeStep, scoreSequence } from "./wasm-bridge"; // WASM bindings

export class TrimSecGuard {
private config: TrimSecConfig;
private latentBuffer: LatentVector[] = [];
private wasmReady = false;

constructor(config?: Partial<TrimSecConfig>) {
this.config = {
sequenceLength: 32,
anomalyThreshold: 0.9,
minSamplesBeforeDecision: 8,
...config,
};
}

async initWasm(wasmUrl: string): Promise<void> {
await initWasmModule(wasmUrl); // loads \& instantiates WASM
this.wasmReady = true;
}

collectFeatures(ctx: SignContext): FeatureVector {
return buildFeatureVector(ctx);
}

async pushStep(features: FeatureVector): Promise<RiskResult> {
if (!this.wasmReady) {
return { anomalyScore: 0, allowed: true, reason: "WASM not ready" };
}

    const latent = encodeStep(features); // calls WASM AE encoder, returns length-4 vector
    
    this.latentBuffer.push(latent);
    if (this.latentBuffer.length > this.config.sequenceLength) {
      this.latentBuffer.shift();
    }
    
    if (this.latentBuffer.length < this.config.minSamplesBeforeDecision) {
      return { anomalyScore: 0, allowed: true, reason: "Insufficient history" };
    }
    
    const padded = this.padSequence(this.latentBuffer, this.config.sequenceLength);
    const score = scoreSequence(padded); // 0..1
    
    const allowed = score < this.config.anomalyThreshold;
    const reason = allowed ? "Normal behavior" : "Anomalous signing pattern";
    
    return { anomalyScore: score, allowed, reason };
    }

private padSequence(seq: LatentVector[], length: number): Float32Array {
const out = new Float32Array(length * 4);
const start = Math.max(0, length - seq.length);
for (let i = 0; i < seq.length; i++) {
const offset = (start + i) * 4;
const z = seq[i];
out[offset]   = z[0];
out[offset+1] = z[1];
out[offset+2] = z[2];
out[offset+3] = z[3];
}
return out;
}

async gateSignRequest<T>(
ctx: SignContext,
signFn: () => Promise<T>
): Promise<T> {
const features = this.collectFeatures(ctx);
const risk = await this.pushStep(features);

    if (!risk.allowed) {
      // Log locally; optionally surface UI prompt
      console.warn("TRIM-SEC guard blocked signing:", risk);
      throw new Error("Signing blocked by TRIM-SEC guard: " + risk.reason);
    }
    
    return signFn();
    }
}
